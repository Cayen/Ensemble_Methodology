Contributing to Ensemble_Methodology

## What This Project Is

This is **live, operational research** into multi-model AI ensemble methodology. 
We are actively monitoring 13 economic vectors with 8 AI models, running daily save states, 
and stress-testing our verification protocols against real-world events.

We are not accepting code contributions. We are actively seeking engagement on methodology.

---

## How to Contribute

### 1. Test the Methodology (In Your Domain)

Apply the verification gate framework to your own work:
- Pick a domain you understand (economics, tech forecasting, policy analysis)
- Implement the six-tag system: `[SOURCED]`, `[CARRIED-FORWARD]`, `[DERIVED]`, `[UNVERIFIED]`, `[ENSEMBLE-DEFINED]`, `[GAP]`
- Document what breaks
- Report back

**How to report:** Open an Issue titled "Methodology test: [your domain]" with:
- What you were monitoring
- Which failure modes you encountered
- What tags caught them, what didn't
- Suggested fixes

### 2. Identify Failure Modes

Have you seen multi-model AI ensembles fail in ways we didn't document?

**Report it as an Issue:**
- Title: "Failure mode: [description]"
- Include: Real example, impact, proposed tag/gate
- We'll evaluate and potentially add it to the framework

### 3. Verify Vectors

We're tracking 13 active economic vectors. If you have:
- Primary source data we haven't found
- Contradictions in our sourcing
- Better data on any vector

**Create an Issue with:**
- Vector name
- Source URL
- Why it matters
- Suggested update to save state

### 4. Cross-Domain Applications

Apply this methodology to completely different fields:
- Climate monitoring ensembles
- Medical diagnosis ensembles  
- Supply chain risk tracking
- Geopolitical analysis

**Report progress as Discussions** (not Issues):
- What's the domain?
- How did you adapt the gate?
- What broke?
- What worked?

---

## What NOT to Do

- **Don't submit pull requests to modify vectors.** We update save states operationally based on daily monitoring.
- **Don't fork for your own ensemble.** Reference and cite, but keep your work separate so we can track what the methodology enables.
- **Don't submit opinions/predictions.** We only integrate sourced, verifiable data.
- **Don't request features.** This methodology is discovered through use, not designed in advance.

---

## What We're Looking For

1. **Independent verification** — People finding/flagging data we missed
2. **Failure mode reports** — Real examples of ensemble reasoning breaking
3. **Cross-domain tests** — Applying this to non-economic prediction
4. **Honest critique** — "This gate doesn't catch X" with examples
5. **Documentation improvements** — Clearer writing, better examples, missing sections
