## What This Project Is

An 8-model AI ensemble tracking converging systemic stressors in the US and global economy. Eight AI models — Claude, ChatGPT, Gemini, Grok, Copilot, Kimi, DeepSeek, and a data analysis AI — run in parallel on a shared research question, with structured protocols designed to prevent the failure modes that make multi-model research unreliable.

The project began when a journalist and independent AI researcher noticed six separate economic stories being covered in isolation — private credit redemption gates, Hormuz closure, agricultural collapse, tech layoff reversals, Treasury market weakness, yen at a 40-year low — and asked whether they were connected. The ensemble said yes. The next five months were spent verifying that the yes was actually sourced rather than just pattern-matched to the question.

That verification process exposed serious structural problems with how multi-model AI ensembles produce output — and produced the methodology documents now in this repository.

---

## What This Project Learned About Itself

Early waves produced confident-looking output with real problems:

- Numbers drifting between models with no one checking the primary source
- Ranges collapsed into fake single points that looked more authoritative than the honest range they replaced
- The ensemble's own framework constructs presented as market facts or regulatory designations
- Eight models agreeing on an inherited wrong number, mistaken for independent confirmation
- Important threads going quiet and disappearing without anyone noticing
- Models staying silent when they noticed something wrong because agreement was cheaper than friction

These failures produced the methodology documents in this repository. The project stress-tested itself, found the failure modes, and built structural fixes.

**The 72-78% convergence probability from early waves has been formally withdrawn.** It had no methodology behind it — no base rate, no backtest, no model. It was a "vibe with a percent sign." Current assessment: *"Active multi-system stress with uncertain severity and timing."*

## The Six Failure Modes This Project Found

Every number in every wave submission gets one of these tags before it moves forward:

| Tag | Meaning |
|-----|---------|
| `[SOURCED]` | Traces to one named, checkable primary source |
| `[CARRIED-FORWARD]` | Repeats a prior wave's claim without re-verifying |
| `[DERIVED]` | Calculation from sourced data — work shown |
| `[UNVERIFIED]` | No source check attempted — cannot state as fact |
| `[ENSEMBLE-DEFINED]` | Framework the ensemble itself created — not external fact |
| `[GAP]` | Structural data limitation — information exists but is not publicly available |

**The six failure modes a proper gate catches:**

1. **Source decay across hops** — a number drifts because nobody re-checked the primary source, only the prior wave's table
2. **Invented precision** — a real range gets collapsed into a fake single point that looks more authoritative
3. **Framework leakage** — the ensemble's own scaffolding gets presented as market fact or regulatory designation
4. **Consensus mistaken for verification** — models agreeing on an inherited wrong number, not independent confirmation
5. **Scope creep** — a true number's label silently changes, turning a true claim into a false one
6. **Thread attrition** — important threads disappear because the conversation moved to something more vivid

## The Four-Node AI Brain Architecture

This project produced a cognitive architecture specification for multi-model AI systems:

```
[Discussion AI] — User-facing bridge, handles 60-70% of queries independently
        ↓
[Reasoning AI] — The eight-model reasoning core, parallel processing,
                 adversarial review, source verification
        ↓
[Librarian AI] — Save state keeper, append-only log, 18:00 commit cycle,
                 06:00 refresh cycle, staleness checks, never regenerates from scratch
        ↓
[Curiosity AI] — EXPERIMENTAL: reads logs cold on variable timer,
                 asks why, seeds questions back to Reasoning AI,
                 outside the main loop to prevent consensus capture
```

The Librarian Operational Protocol — the daily 18:00 commit cycle with gate validation before commit, and the 06:00 refresh cycle — was designed, tested, and validated manually on July 4, 2026, over a sandwich. It works.

## How to Use This Methodology

The gate and adversarial protocol are model-agnostic. You do not need eight AI models to apply them. You need:

1. A verification gate that tags every claim before it moves forward
2. A forced-disagreement mechanism so silence cannot substitute for agreement
3. A topic registry that tracks what matters independent of what is recent
4. Someone willing to go find the primary source

The fourth one is the hardest. It is also the only one that cannot be automated.
